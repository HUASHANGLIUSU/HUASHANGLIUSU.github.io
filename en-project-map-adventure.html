Godot · Exploration · Event-driven Adventure

Map Adventure Game

Evolved from a C++ console prototype through a Qt Widgets desktop interface to a Godot version, linking fog-of-war exploration with events, combat, shops, camps, progression, and collectibles.

GodotGDScriptC++ / QtFog of WarEvents & Combat

Core Loop

The player leaves camp for a layered map, chooses routes inside the visible area, and spends action resources. Tiles trigger combat, supplies, shops, or special events, feeding back into health, threat, inventory, progression, and collection goals.

Map System

  • Grid map, floor progression, and route selection
  • Per-tile visibility rules and fog of war
  • Synchronized exploration, resource, and threat states
  • Risk-reward pacing through event tiles

Gameplay Modules

  • Conditional event choices
  • Turn-based combat and tide presentation
  • Shop, camp, inventory, and progression
  • Collectibles, records, and action logs

Engineering Evolution

The project preserves the original C++ game logic while adding a Qt Widgets desktop migration and a Godot/GDScript rebuild. Behavior checks cover shop parity, map visibility, targeting blockers, events, and state transitions.
